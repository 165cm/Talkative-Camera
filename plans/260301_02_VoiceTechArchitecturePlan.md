# 音声認識・レスポンス改善：3つのアーキテクチャプラン（コスト重視）

現在のコード（`src/services/gemini.ts`）を分析したところ、現状は **「Gemini 2.5 Flash Native Audio (Live API)」** という最新の技術が使われていました。これは本来非常に高速な部類ですが、以下の理由で「遅い」「聞き取らない（特に英語）」という現象が起きています。

- **英語を聞き取らない理由**: システムプロンプト（AIへの命令）が日本語で書かれているため、Geminiの「耳」が日本語モードになっており、日本人訛りの英語を弾いてしまっている可能性が高いです。他の英会話アプリは明示的にSTT（音声認識）を「英語（en-US）」に固定しているため正確に聞き取ります。
- **遅い理由**: Live APIの「ユーザーが話し終わった」と判定する沈黙時間（VAD閾値）が長いため、結果的にレスポンスが遅く感じられます。

これらを踏まえ、**「できるだけコストを抑える（無料〜激安）」** かつ **「レスポンスと精度を上げる」** ための3つのプランを提案します。

---

## プランA：【完全無料・爆速】ブラウザネイティブ構成
一番お金がかからず、一番レスポンスが早い構成です。

- **STT (音声認識)**: Web Speech API（ブラウザ標準機能）
  - 明示的に `lang = 'en-US'`（英語）に固定可能。**英語の認識精度が跳ね上がります。**
- **LLM (AI思考)**: Groq (Llama-3系) または 現状のGemini 1.5 Flash（テキストのみ利用）
  - Groqは現在AI界で最もレスポンスが早い（ほぼ0秒で返答が来る）APIです。無料枠もあります。
- **TTS (音声合成)**: Web Speech API（ブラウザ標準）
- **コスト**: **0円 / 回**（完全無料）
- **メリット**: テンポが圧倒的に早い。お金が一切かからない。
- **デメリット**: 声の種類（合成音声）がブラウザ依存になるため、キャラクターごとの声の演じ分け（Puck, Charonなど）が現状よりチープになります（ロボット感が強まる）。

## プランB：【高精度・ハイブリッド】Whisper採用構成
英会話アプリと同等の「聞き取り能力」を持たせつつ、コストを極限まで下げる構成です。

- **STT (音声認識)**: Groq Whisper API (whisper-large-v3)
  - 世界最高精度の音声認識。しかもGroq経由なら爆速でテキスト化されます。
- **LLM (AI思考)**: Groq (Llama-3系) または Gemini 1.5 Flash
- **TTS (音声合成)**: 現在のGemini (テキスト→音声) または OpenAI TTS (tts-1)
- **コスト**: **約0.1円 〜 0.3円 / 1会話（1往復）**
  - （※Whisperが1分 約0.06円、LLMがほぼ0円、TTSが1回約0.2円）
- **メリット**: **英語の聞き取り精度が最強**になります。日本人の英語でも正確にテキスト化します。
- **デメリット**: 完全に無料ではありません。フロントエンドの録音ロジックを少し書き換える必要があります。

## プランC：【現状維持＋プロンプト改修】Gemini Live API チューニング構成
現在のシステム（Live API）のまま、コストをかけずに弱点を克服する構成です。

- **システム**: 全て Gemini 2.5 Flash Native Audio (現在のまま)
- **改修内容**:
  1. **プロンプトの英語化**: ユーザーが「英語」で話しかける設定の場合、システムプロンプト自体を完全に英語で記述し、Geminiの耳を強制的に「英語モード」にします。（これだけで精度が劇的に改善するはずです）
  2. **UIでの「送信ボタン」追加**: 沈黙を待つのが遅い原因なので、「話し終わったらタップして送信」するUI（トランシーバー形式）を追加し、強制的にターンを終了させます。
- **コスト**: **現状通り（ほぼ0円/API無料枠内、または 0.1円以下/回）**
- **メリット**: 今のコードベースを大きく変えずにすぐ実装可能。声のクオリティ（Puckなど）を維持できる。
- **デメリット**: Gemini側の仕様に依存するため、プランAやBほどの「圧倒的な爆速感」にはならない可能性があります。

---

### おすすめのアプローチ
一番手軽で声のクオリティが落ちないのは **【プランC】** です。
もしプランCでプロンプトを改修しても「まだ英語の精度が不満！」となった場合は、STTだけを超高精度なものに入れ替える **【プランB】** に移行するのが安全かつ高品質です。

（※もし「声がチープになってもいいから、完全無料でとにかく爆速のキャッチボールがしたい！」場合は **【プランA】** になります。）
