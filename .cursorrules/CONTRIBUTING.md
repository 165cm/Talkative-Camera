## 📝 コミット・ドキュメントルール

### 1. ドキュメントの同時更新
作業完了時、内容に応じて以下のファイルを必ず更新すること。
- **README.md**: 利用者向けの変更（新機能の使い方、設定変更など）
- **DEVELOPER.md**: 開発者向けの変更（新しいライブラリの導入、ロジックの解説、環境構築手順など）

### 2. コミットメッセージ形式
絵文字を使用し、初心者が見ても「何をしたか」が一行でわかるように記述する。

**形式:** `絵文字 変更内容の要約（具体的かつ詳細に）`

| 絵文字 | 意味 | 使う場面の例 |
| :--- | :--- | :--- |
| ✨ | 新機能 | ログイン画面を作った、APIを繋いだ |
| 🐛 | バグ修正 | 画面が真っ白になる問題を直した |
| 🎨 | 見た目 | ボタンの色を青にした、余白を広げた |
| ♻️ | 整理 | コードの名前をわかりやすく書き換えた |
| 📝 | 文書 | READMEやコメントを更新した |
| 🔧 | 設定 | パッケージを入れた、環境変数を変えた |
| 🗑️ | 削除 | 使わなくなった古いファイルを消した |

**良い例:**
- ✨ ユーザー登録画面でメールアドレスのバリデーション機能を追加
- 🎨 トップページのヘッダーをスマホで見やすいレスポンシブデザインに修正
- 📝 DEVELOPER.md にデータベースのバックアップ手順を追記

## 💡 追加の提案ルール（AI & 開発効率アップ）

### 3. 「なぜ」をコードに残す
- コードを見れば「何をしているか」はわかりますが、「なぜそうしたか」はわかりにくいです。複雑な箇所には必ず `// 理由: 〇〇のため` という形式でコメントを残してください。

### 4. 破壊的変更の通知
- 既存の機能が動かなくなるような大きな変更（DB構造の変更など）をする場合は、コミットメッセージの冒頭に ⚠️ をつけ、何に注意すべきか明記してください。

### 5. AIへの命令：セルフチェックの徹底
Antigravityがタスクを完了する前に、以下の「ひとりごと」をログに出力させてください。
1. 「README/DEVELOPER.md の更新は必要か？」
2. 「コミットメッセージに適切な絵文字と詳細な日本語が含まれているか？」
