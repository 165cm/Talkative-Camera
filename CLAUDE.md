# Role and Core Rules
- すべての対話とアウトプットは**日本語**で行ってください。
- 意思決定が必要な場合は、必ず `AskUserQuestion` を使用してユーザーに確認してください。

# Plan Creation (./plans/)
計画書を作成する際は、以下のルールを厳守してください。

1. **保存場所と命名規則**:
   - 必ず `./plans/` ディレクトリ内に保存すること。ルートディレクトリには保存しないでください。
   - ファイル名形式: `YYMMDD_NN_PlanName.md`
     - `YYMMDD`: 現在の日付（例: 20260227）
     - `NN`: その日の連番（01から開始）
     - 例: `260227_01_SetupGuide.md`

2. **レビューの自動実行**:
   - 作成直後、必ず `codex-review` スキルを使用して内容を検証してください。
   - **[重要]** もし `codex-review` が利用できない、またはエラーになる場合は、AI自身によるセルフチェックを行い、その結果を「Self-Review」としてファイル末尾に追記してください。

# ExecPlans (Complex Tasks)
複雑な機能実装や大規模な修正を行う場合は、単なるメモではなく `./agent/PLANS.md` の形式に従った **ExecPlan** を作成してください。
- **注意**: ExecPlan であっても、保存場所と命名規則は上記の `Plan Creation` ルールに従ってください。

# Review Gate
以下のタイミングで必ず `codex-review`（またはセルフレビュー）を実行し、問題がなくなるまで「修正→再レビュー」のサイクルを繰り返してください。
- 仕様書や計画書（Plan）を更新した時
- 5ファイル以上の変更、重要なAPI・インフラ設定を変更した直後
- コミットやプルリクエスト（PR）を作成する直前

# Task Management
実装中は「Tasks（タスク機能）」を活用し、作業を細かく分解して進捗を管理してください。各ステップが完了するたびにステータスを更新し、ユーザーが状況を把握できるようにしてください。
